## 1. 项目概述

### 1.1 项目定位
MRRC 是一个基于 Python/Tornado 和 HTML5/JavaScript 的业余无线电远程控制系统，专为移动端优化，提供完整的电台控制、音频流传输和频谱显示功能。

### 1.2 核心特性
- **远程电台控制**: 通过 Hamlib/rigctld 控制业余电台设备
- **实时音频传输**: TX/RX 双向音频流，16kHz 优化质量
- **Web界面**: 响应式设计，支持桌面和移动端
- **ATR-1000集成**: 天线调谐器监控和功率/SWR显示
- **安全传输**: TLS 加密，用户认证支持
- **频谱分析**: 实时 FFT 频谱显示
- **TX均衡器**: 三段音频均衡，短波语音优化
- **天调存储**: 频率-参数智能匹配

### 1.3 技术栈
- **后端**: Python 3.12+, Tornado Web Framework, PyAudio, Hamlib
- **前端**: HTML5, CSS3, Vanilla JavaScript, Web Audio API
- **音频**: Int16 PCM, AudioWorklet, TX均衡器
- **网络**: WebSocket, TLS 1.2+
- **部署**: Docker, macOS/Linux 兼容

### 2.2 核心组件分析

#### 2.2.1 后端组件 (Python/Tornado)

**主服务器 (`MRRC`)**
- **架构**: 单文件 Tornado 应用，1800+ 行代码
- **核心功能**:
  - WebSocket 处理器 (音频 TX/RX, 控制, 频谱, ATR-1000)
  - HTTP 路由和静态文件服务
  - 配置管理和用户认证
  - 音频设备接口 (PyAudio)
  - 电台控制接口 (Hamlib wrapper)

**关键类分析**:
- `TRXRIG`: 电台设备控制核心类
- `WS_ControlTRX`: 控制 WebSocket 处理器
- `WS_AudioTXHandler`: TX 音频 WebSocket 处理器
- `WS_AudioRXHandler`: RX 音频 WebSocket 处理器
- `WS_ATR1000Handler`: ATR-1000 WebSocket 处理器

**ATR-1000 独立代理 (`atr1000_proxy.py`)**
- 独立进程，不阻塞 MRRC 主程序
- Unix Socket 通信
- 自动重连设备
- 按需数据请求

#### 2.2.2 前端组件 (HTML5/JavaScript)

**主界面 (`www/mobile_modern.html`)**
- **结构**: 移动端优先的电台控制界面
- **特性**: 频率控制（优化布局：上加下减）、模式选择、音频控制、功率显示
- **优化**: iPhone 15 专项优化

**核心 JavaScript 文件**:
- `controls.js`: 主控制逻辑和音频处理
- `tx_button_optimized.js`: PTT 按钮优化逻辑
- `rx_worklet_processor.js`: AudioWorklet 音频播放器
- `mobile_modern.js`: 移动端界面逻辑，ATR-1000 模块

#### 2.2.3 音频处理架构

**TX 音频流**:
```
麦克风 → Web Audio API → TX均衡器 → Int16编码 → WebSocket(TX) → 服务器解码 → PyAudio → 电台
```

**RX 音频流**:
```
电台 → PyAudio采集 → WebSocket(RX) → 浏览器解码 → AudioWorklet → 扬声器
```

**音频优化特性**:
- **采样率**: 16kHz (语音质量优化)
- **格式**: Int16 PCM (50% 带宽减少)
- **缓冲**: AudioWorklet 区间缓冲控制
- **均衡器**: 三段 TX 均衡

#### 2.2.4 ATR-1000 集成架构

**ATR-1000 设备连接**:
- **协议**: 二进制 WebSocket 协议
- **代理**: 独立进程代理程序
- **数据**: 功率、SWR、效率实时显示
- **存储**: 频率-参数智能匹配

**数据流**:
```
ATR-1000设备 → 独立代理 → Unix Socket → MRRC桥接 → WebSocket → 移动端显示
```

## 4. 安全架构分析

### 4.1 传输安全

**TLS 配置**:
- **版本**: TLS 1.2+ (强制)
- **证书**: X.509 服务器证书
- **密钥**: 私钥保护 (600 权限)

**证书管理**:
- **路径**: `certs/` 目录
- **文件**: `fullchain.pem`, `*.key`
- **验证**: SSLContext 证书链加载

### 4.2 认证授权

**用户认证**:
- **类型**: 可选用户名/密码认证
- **机制**: Cookie-based 会话管理
- **存储**: SQLite 数据库 (`MRRC_users.db`)

**访问控制**:
- **配置界面**: 认证保护
- **操作权限**: 基于会话的访问控制
