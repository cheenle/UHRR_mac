MRRC V5.4 Architecture

SDD-MRR-2026-001 · IBM TeamSD v2.3.2 · Mobile Remote Radio Control · BG1SB/ON80

AD-001: Tornado Web Framework

  • • Native WebSocket support, async I/O
  • • Compatible with PyAudio event loop
  • • Single-threaded async model
  • • Mature, stable, active community

AD-002: Int16/Opus Codec

  • • Opus: 16kHz, 28kbps, 20ms frames
  • • WebRTC standard, native browser support
  • • Int16 PCM for compatibility fallback
  • • Low latency, high compression ratio

AD-003: WDSP NR2 DSP

  • • OpenHPSDR professional DSP library
  • • NR2 spectral noise reduction for SSB
  • • NB, ANF, AGC modules included
  • • ~20ms processing latency at 48kHz

AD-004: Multi-Instance

  • • Independent config files per instance
  • • Unix Socket isolation (no port conflicts)
  • • mrrc_multi.sh + Docker Compose
  • • Minimal code changes, reuses architecture

AD-005: PTT Reliability

  • • Warm-up frames fill switching gap
  • • Retry 3x at 50ms intervals
  • • ~50ms additional latency (acceptable)
  • • 99%+ press-to-transmit success rate

AD-006: Audio Buffer

  • • 256-sample buffer at 48kHz = 5.3ms
  • • Adaptive sizing for network conditions
  • • AudioWorklet client-side buffering
  • • Balance latency vs. reliability

AD-007: CW ONNX Decode

  • • 2MB ONNX model in browser
  • • <50ms inference latency
  • • Zero server-side dependencies
  • • PyTorch+CUDA backend fallback

AD-008: ATR Proxy

  • • Independent proxy process (exclusive device)
  • • Unix Socket bridge (low overhead)
  • • REST API :8080 for 3rd-party
  • • Dynamic polling: idle 15s / active 5s / TX 0.5s
