## 安装步骤

### 1. 安装Python依赖

```bash
cd /Users/cheenle/UHRR/MRRC

# 安装Whisper和其他依赖
pip3 install openai-whisper sounddevice soundfile tornado numpy

# 可选：安装Piper TTS（更高质量的语音合成）
pip3 install piper-tts
```

### 2. 启动语音助手服务

```bash
# 使用启动脚本
./start_voice_assistant.sh

# 或手动启动
python3 voice_assistant_service.py --port 8878 --whisper-model base --language zh
```

参数说明：
- `--port`: 服务端口，默认8878
- `--whisper-model`: 模型大小 (tiny/base/small/medium/large)，默认base
- `--language`: 识别语言，默认zh(中文)

### 3. 访问语音文字助手

1. 启动MRRC主服务
2. 打开移动端界面 `mobile_modern.html`
3. 点击菜单 → 🎙️ **语音文字助手**
4. 或使用直接访问: `http://<服务器IP>:<MRRC端口>/mobile_voice_text.html`

## 使用方法

### 语音识别（电台→文字）

1. 点击**"启动"**按钮开始识别
2. 后端自动从声卡采集音频
3. 识别结果实时显示在"电台接收识别"区域
4. 识别内容自动添加到对话记录

### 语音合成（文字→电台）

1. 在"发送回复"框输入文字
2. 可选择语速（慢/正常/快）
3. 点击**"按住发射"**按钮发送
4. 后端合成语音并通过电台发射

### 快捷功能

- **⚡ 快捷**: 预设常用业余无线电用语
- **🕐 历史**: 查看和快速选择发送历史
- **▶️ 试听**: 使用浏览器TTS试听（不发射）
- **⏹️ 停止**: 停止当前合成和发射

## 模型选择

| 模型 | 大小 | 速度 | 准确率 | 适用场景 |
|------|------|------|--------|----------|
| tiny | 39 MB | 最快 | 一般 | 测试/低配置设备 |
| base | 74 MB | 快 | 良好 | 日常使用推荐 |
| small | 244 MB | 中等 | 好 | 高准确率需求 |
| medium | 769 MB | 慢 | 很好 | 专业使用 |
| large | 1550 MB | 很慢 | 最好 | 最高准确率 |

**推荐**: base模型在准确率和速度间有良好平衡。

## 故障排除

### 服务启动失败

```bash
# 检查端口是否被占用
lsof -i :8878

# 检查依赖是否安装
pip3 list | grep -E "whisper|sounddevice|tornado"
```

### 无法识别语音

1. 检查声卡输入设备是否正确
2. 检查系统音量是否开启
3. 查看后端日志是否有错误

### WebSocket连接失败

1. 确认服务已启动: `curl http://localhost:8878/api/status`
2. 检查防火墙设置
3. 确认端口可访问

### 语音合成失败

1. 如果没有Piper TTS，会使用pyttsx3（系统TTS）
2. 检查系统音频输出是否正常
3. 查看后端日志

## 配置文件

可以设置环境变量：

```bash
export VOICE_PORT=8878          # 服务端口
export WHISPER_MODEL=base       # 模型大小
export LANGUAGE=zh              # 识别语言
```

## 与MRRC集成

语音助手服务是独立运行的，但可以通过以下方式与MRRC集成：

1. **频率同步**: 自动显示当前电台频率
2. **PTT控制**: 按住发射按钮同时控制MRRC的PTT
3. **状态共享**: 通过postMessage与主页面通信

## 技术栈

- **后端**: Python 3.8+, Tornado, OpenAI Whisper
- **前端**: HTML5, CSS3, Vanilla JS
- **通信**: WebSocket
- **音频**: PyAudio, SoundDevice
