# 架构决策记录 (ADR) 模板

> 每个重要架构决策都应该有明确的记录

---

## ADR 模板

### AD-XXX: [决策标题]

| 属性 | 值 |
|------|-----|
| ID | AD-XXX |

| 状态 | 提议 / 已批准 / 已废弃 / 已替换 |
| 决策者 | 名字 |
| 影响范围 | 模块/组件 |

[清晰描述这个决策要解决的核心问题]

- 相关的业务需求

- 可用资源
]

#### 决定

[清晰描述最终的决定]

- 技术优势
- 成本效益
- 风险考量
]

#### 确认

---

## 决策清单

使用此清单确保决策完整：

- [ ] 问题清晰定义
- [ ] 背景信息完整
- [ ] 至少考虑3个替代方案
- [ ] 决策理由充分
- [ ] 影响分析完整
- [ ] 派生需求记录
- [ ] 相关团队已知会
- [ ] 文档已版本化

---

## 常见决策类型

### 技术选型
- 编程语言
- 框架选择
- 数据库选型
- 云服务提供商

### 架构模式
- 微服务 vs 单体
- 同步 vs 异步
- REST vs GraphQL

### 基础设施
- 部署策略
- 容器化方案
- 监控方案

### 安全
- 认证方案
- 授权策略
- 数据加密
