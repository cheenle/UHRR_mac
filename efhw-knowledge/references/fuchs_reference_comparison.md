# Fuchs 拓扑 EFHW 调谐器 — 参考设计对比分析

> 对比 OE1JF (原始), M0UKD, F5NPV, 和 BG1SB V3.0 四个 Fuchs 派生设计
> 日期: 2026-06-09

---

## 1. 设计来源谱系

```
OE1JF Fuchs (1960s, 奥地利)
  原始并联 LC 谐振耦合器, 空气芯电感+可变电容
  │
  ├─ M0UKD (2010s, 英国)
  │   量化分析+实测数据, 2:16T 铜管空气芯, 1200pF
  │   https://m0ukd.com/homebrew/baluns-and-ununs/end-fed-half-wave-antenna-tuned-coupler-efhw/
  │
  ├─ F5NPV (2021, 法国)
  │   ESP8266 WiFi 控制 + Arduino Nano SWR桥
  │   伺服驱动 1200pF 可变电容, 40m-20m
  │   https://f5npv.wordpress.com/remote-automatic-atu/
  │
  └─ BG1SB V3.0 (2026, 中国) ← 本设计
      ESP32-S3 单芯片, T200-6 磁环替代空气芯
      10-500pF 伺服, 40m-10m WARC 全覆盖
      MRRC WebSocket 集成, SWR 委托 ATR1000
```

## 3. 电感设计深度对比

### 3.1 OE1JF / M0UKD / F5NPV — 空气芯

```
构造: 4mm 铜管绕制, ~31mm 直径, ~120mm 长, 16 匝
优点:
  - 零磁芯损耗 → 极高效率 (>98%)
  - 大功率无饱和风险
  - 可承受极高电压/电流
缺点:
  - 体积大, 占用空间
  - 匝间电容不可控 (高频段 Q 下降)
  - 需要铜管和弯管工具
  - 机械不稳定 (振动可能改变电感)

F5NPV 实测: 40m-20m 效率良好, 10m 无法调谐 (Q 过高)
```

### 3.2 BG1SB V3.0 — T200-6 磁环

```
构造: 0.8mm/0.5mm 漆包线绕制在 T200-6 (OD=50.8mm)
优点:
  - 紧湊 (OD 51mm vs 空气芯 120mm+)
  - 机械稳定 (扎带固定)
  - Type 6 μ=8 在 10-30MHz 有 Q>150
  - AL 精确 (10.5 nH/N²) → 电感量可预测
  - 制作简单 (普通漆包线即可)
缺点:
  - 存在磁芯损耗 (但 B_peak=12.7mT 极低, 损耗可忽略)
  - 有 B_sat 上限 (校核: 47× 裕度 → 安全)
  - 匝间电容略高于空气芯 (但 Type 6 低频损耗低)

调谐范围: 5.0-35.1 MHz → 40m-10m 全覆盖
原因: 较小的电容范围 (10-500pF vs 1200pF) 降低了 Q 值范围,
      使谐振在更宽频率范围内可调。
```

## 4. 电容选择对比

| | M0UKD | F5NPV | V3.0 |
|---|-------|-------|------|
| **容值** | 1200pF | 1200pF | 10-500pF |
| **驱动** | 手动 | 伺服 (MG996R) | 伺服 (MG996R) |
| **连续** | 是 | 是 | 是 |
| **覆盖范围** | 80m-20m | 40m-20m | **40m-10m** |

### 为什么 V3.0 用 500pF 而非 1200pF?

```
L=2.06µH (固定)

C=1200pF: f_min = 1/(2π√(2.06µH × 1200pF)) = 3.2 MHz (覆盖 80m)
           f_max = 1/(2π√(2.06µH × 10pF)) = 35.1 MHz (覆盖 10m)
           问题: Q = ωL/R @ 3.2MHz 太低, 80m 效率很差

C=500pF:  f_min = 1/(2π√(2.06µH × 500pF)) = 5.0 MHz (40m OK)
           f_max = 35.1 MHz (10m OK)
           优势: Q 范围收窄, 全频段效率均衡

结论: 1200pF 的 '多出来' 的覆盖 (80m) 在实际中效率太低, 
      不值得为此牺牲 10m 的 Q 值。500pF 是 40m-10m 的 sweet spot。
```

## 5. 控制架构对比

| | F5NPV | V3.0 |
|---|-------|------|
| **主控** | ESP8266 (WiFi Web) | **ESP32-S3 (WiFi+WS)** |
| **副控** | Arduino Nano (SWR采样+伺服) | **无** (ESP32-S3 单芯片全包) |
| **MCU 间通信** | UART | N/A |
| **故障点** | 2 MCU + UART | **1 MCU** |
| **调谐算法** | Arduino 全扫描 | **ESP32-S3 缓存+粗扫+细扫** |
| **远程协议** | HTTP (ESP8266 Web Server) | **WebSocket JSON (持久连接)** |
| **系统集成** | 独立 (自建 WiFi 热点或 STA) | **MRRC 深度整合 (子面板)** |

## 9. 设计教训 (从前人设计中学到的)

### 从 F5NPV 学到的
- 伺服调谐后必须断电 → 消除 PWM 噪音对接收的影响 ✓ (V3.0 用 MOSFET)
- 40m-20m 限制是 Fuchs 拓扑的物理规律 → V3.0 用 500pF (非 1200pF) 突破到 10m
- 双 MCU 是不必要的复杂度 → V3.0 合并为单 ESP32-S3

### 从 M0UKD 学到的
- 空气芯铜管虽然高效但体积大 → V3.0 改用 T200-6 (紧凑 10×)
- 定量 L/C 分析是设计的前提 → V3.0 SDD §2 含完整谐振验证

### 从 OE1JF 学到的
- 并联 LC 谐振原理是永恒的 → V3.0 的数学基础与 1960s 完全相同
- 好的天线 + 好的匹配 = 不需要纯"天调" → V3.0 同样强调天线本身必须谐振
